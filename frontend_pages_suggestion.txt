================================================================
  WeLaund — Frontend Pages Suggestion
  Based on backend controllers and database schema
================================================================

----------------------------------------------------------------
  SHARED / PUBLIC
----------------------------------------------------------------
1. Login Page
   Path    : frontend/public/index.php
   Elements: Role-selector tabs (Super Admin / Owner / Staff / Customer)
             Email/Username + Password fields
             Login button
             Link to Register

2. Register Page
   Path    : frontend/public/register.php
   Elements: Shop selection dropdown (fetches active laundry_shops)
             First Name, Middle Name, Last Name fields
             Contact Number, Address, Email, Password fields
             Submit button (account set to Pending until staff approves)

----------------------------------------------------------------
  SUPER ADMIN MODULE
----------------------------------------------------------------
3. Dashboard
   Path    : frontend/super_admin/dashboard.php
   Elements: Stat cards — Total Shops, Active Shops, Total Owners,
             Total Staff, Total Customers, Pending Customers,
             Total Orders, Total Revenue
             (data from SuperAdminController::getPlatformStats())

4. Shops Management
   Path    : frontend/super_admin/shops.php
   Elements: Table — Shop Name, Address, GCash Number, Status, Created On
             Create Shop + Owner modal (shop name, address, owner name, email, password)
             Activate / Deactivate toggle per row
             (SuperAdminController::getAllShops(), createShopWithOwner(), setShopStatus())

5. Owners Management
   Path    : frontend/super_admin/owners.php
   Elements: Table — Full Name, Email, Shop Name, Status
             Activate / Deactivate toggle
             (SuperAdminController::getAllOwners(), setOwnerStatus())

6. Staff Management (Global)
   Path    : frontend/super_admin/staff.php
   Elements: Table — Full Name, Email, Shop Name, Status
             Activate / Deactivate toggle
             (SuperAdminController::getAllStaff(), setStaffStatus())

7. Customers Management (Global)
   Path    : frontend/super_admin/customers.php
   Elements: Table — Full Name, Email, Shop Name, Status badge
             Status toggle (Approved / Inactive / Pending)
             (SuperAdminController::getAllCustomers(), setCustomerStatus())

----------------------------------------------------------------
  OWNER MODULE
----------------------------------------------------------------
8. Dashboard
   Path    : frontend/owner/dashboard.php
   Elements: Stat cards — Total Customers, Monthly Income, Yearly Income
             Monthly income bar/line chart
             Yearly income summary table
             (OwnerController::getMonthlyIncome(), getYearlyIncome())

9. Staff Management
   Path    : frontend/owner/staff.php
   Elements: Table — Full Name, Email, Status, Created On
             Create Staff form (first name, last name, email, password)
             Edit Staff modal
             Active / Inactive toggle
             (OwnerController::getAllStaff(), createStaff(), updateStaffStatus())

10. Services Management
    Path    : frontend/owner/services.php
    Elements: Table — Service Name, Unit (per_kg / per_piece), Price, Status
              Add Service form
              Edit Service modal
              Active / Inactive toggle
              (OwnerController::getServices(), createService(), updateService(), updateServiceStatus())

11. Shop Settings
    Path    : frontend/owner/settings.php
    Elements: Edit form — Shop Name, Address, GCash Name, GCash Number
              Save button
              (OwnerController::getShop(), updateShopSettings())

----------------------------------------------------------------
  STAFF MODULE
----------------------------------------------------------------
12. Dashboard
    Path    : frontend/staff/dashboard.php
    Elements: Daily income total card
              Daily customer requests list (auto-resets Done orders)
              Quick links to Orders and Customer Approvals
              (StaffController::getDailyTotal())

13. Customer Approvals
    Path    : frontend/staff/customers.php
    Elements: Table of Pending customers — Full Name, Email, Contact, Date Applied
              Approve / Disapprove buttons per row
              (StaffController::getPendingCustomers(), updateCustomerStatus())

14. Orders List
    Path    : frontend/staff/orders.php
    Elements: Table of Ongoing orders — Order Ref, Customer Name, Contact, Date
              View Details button per row
              (StaffController::getActiveOrders())

15. New Order
    Path    : frontend/staff/new_order.php
    Elements: Customer selector dropdown (Approved customers only)
              Service + weight/quantity rows (add multiple)
              Auto-calculated subtotal and total amount
              Pickup/Delivery type selector
              Submit to create order
              (StaffController::getApprovedCustomers(), getServices(), createOrder())

16. Order Details
    Path    : frontend/staff/order_details.php
    Elements: Order info — Ref, Customer name, address, contact
              Items table — Service, Unit, Weight/Qty, Subtotal
              Total weight and total amount
              Order status update buttons (Ongoing / Done / Cancelled)
              Payment status badge (Paid / Unpaid)
              Link to Payment page
              (StaffController::getOrder(), updateOrderStatus())

17. Payment
    Path    : frontend/staff/payment.php
    Elements: Payment method toggle — GCash or Manual Cash
              GCash: transaction reference input + amount + Verify button
              Manual: amount input + mark as Paid button
              Shop GCash name/number display for reference
              Receipt preview on success
              (StaffController::verifyGcashPayment(), getShopGcashDetails())

----------------------------------------------------------------
  CUSTOMER MODULE
----------------------------------------------------------------
18. Dashboard
    Path    : frontend/customer/dashboard.php
    Elements: Account status banner (Pending / Approved / Disapproved)
              Active order card with current status
              Notification area (e.g., "You can now bring your clothes to the shop")
              Quick links to Request Order and History

19. Request Order
    Path    : frontend/customer/new_order.php
    Elements: Available services list (read-only, for reference)
              Payment method selection (GCash / Manual)
              Submit request button
              Note: actual weight/pricing is set by staff

20. Order History
    Path    : frontend/customer/history.php
    Elements: Table — Order Ref, Date, Total Amount, Payment Status, Order Status
              View Receipt button per completed order

21. Receipt View
    Path    : frontend/customer/receipt.php
    Elements: Shop name and details
              Order reference and date
              Items breakdown (service, weight/qty, subtotal)
              Total weight, total amount, discount
              Payment method and transaction reference
              Payment status badge

22. Profile
    Path    : frontend/customer/profile.php
    Elements: Edit form — First Name, Middle Name, Last Name
              Contact Number, Address (read-only or editable)
              Change Password section (current password, new password, confirm)
              Save button

----------------------------------------------------------------
  IMPORTANT NOTES
----------------------------------------------------------------
- Login page MUST include a role selector since AuthController::login()
  requires a $role parameter (super_admin, owner, staff, customer)

- Register page MUST fetch active laundry_shops for the shop dropdown
  since customers are bound to a specific shop

- Customer dashboard MUST show a status banner for Pending/Disapproved
  accounts since login is blocked for non-Approved customers

- Staff payment page handles two flows:
    GCash  -> verifyGcashPayment() inserts into payments table
    Manual -> direct payment_status update on orders table

- Owner dashboard charts use:
    getMonthlyIncome() -> grouped by year + month
    getYearlyIncome()  -> grouped by year

- All role-protected pages must call AuthController::requireRole()
  at the top to enforce RBAC
